# DDF Schema Reference

Canonical reference for DDF (Device Definition File) sections and their fields.
Source: `DeviceLib.pdf` (37 pages) + pilot DDF analysis.

> Items marked `// AMBIGUOUS` need confirmation from Martin.

## File Format

- CSV with `;` delimiter
- Encoding: UTF-8 or CP1252 (auto-detected via BOM + chardet)
- Lines starting with `#` are comments (skipped by parser)
- Sections start with `*SECTION_NAME`
- Cells containing `;` must be wrapped in `"` quotes (e.g., formulas)
- The last character inside formula quotes must be `;`

## Section Overview

Per DeviceLib.pdf p.5, the sections are:

| Section | Alternative Name | Purpose |
|---------|-----------------|---------|
| `*GENERAL` | — | Device metadata + connection/auth parameters |
| `*COMMAND` | — | User-triggered commands |
| `*READ` | `*PREPROCESS` | Polling-based data reads |
| `*WRITE` | `*ONCHANGE` | Event-triggered writes (on change/force) |
| `*LISTENER` | — | Socket-based event reception (REST) |
| `*GROUP` | — | Sub-device grouping |
| `*ITEM` | — | Logical variable definitions |
| `*OBJECT` | — | Generic device UI elements |
| `*IO` | — | I/O signal definitions |
| `*CONFIG` | — | User-configurable parameters (max 3, retain) |
| `*ARGS` | — | Sub-section of READ/WRITE/LISTENER |
| `*SIGNATURE` | — | Cryptographic signature (added by DeviceLib DB) |
| `*FUNCTION` | — | MODBUS-only function definitions |

## Sections in Detail

### `*SIGNATURE`

First section in signed DDFs. Added by the DeviceLib database on download.
Per DeviceLib.pdf p.4: "Die Files werden beim Downloaden aus der DB signiert."

| Field | Type | Description |
|-------|------|-------------|
| SIGN_ALGO | string | Signature algorithm (`ECDSA-SHA384`) |
| SIGN_DATE | datetime | Signing timestamp |
| FILE_VERDATE | string | File version hash + date |
| SIGNATURE | string | Base64-encoded ECDSA signature |

### `*GENERAL`

Appears in two logical blocks in the same section:

**Block 1 — Device Identity** (DeviceLib.pdf p.6)

| Field | Type | Description |
|-------|------|-------------|
| DEVICE | string | Device name |
| MANUFACTURER | string | Manufacturer name |
| TYPE | string | Device type (e.g., `Gateway`, `Air conditioner`, `Ventilation unit`) |
| PROTOCOL | string | `SERCOM`, `MODBUS TCP`, `MODBUS ASCII`, `MODBUS RTU`, `REST-API (DDF)`, `DMX512` |
| MODEL_NR | string | Model number |
| VERSION_NR | string | Version number |
| ID | hex string | Unique device ID — combination of protocol + model + version per manufacturer |
| MIN_CONTROL_VERSION | string | Minimum controller firmware version (e.g., `V9615-08`) |
| TIMESTAMP | datetime | Creation date (generated by database) |
| VERSION_INFO | string | Optional version info |
| VERSION_USER | string | User who created version |
| REVISION | string | Revision hash + date |

**Block 2 — Connection Parameters** (DeviceLib.pdf p.6-8)

General (all protocols):

| Field | Type | Description |
|-------|------|-------------|
| CONNECTION | enum | `SERIALPORT`, `SERIALPORT+`, `IPADDRESS`, `DOMAIN`, `DEFDOMAIN` |
| SLAVESMAX | int | Max sub-devices configurable via UI. Missing = 1 device only |
| SLAVE | string | Default slave address for first device |
| DEBUGPORT | int | TCP port for debug output. **Must be 8500-8600** |

REST-specific:

| Field | Type | Description |
|-------|------|-------------|
| DOMAIN | string | API domain (URL or IP) |
| AUTHENTIFICATION | enum | `NONE`, `PASSWORD`, `DEFPASSWORD`, `OAUTH_DEVICE_FLOW`, `OAUTH_GRANT_FLOW` |
| PASSWORD | string | Password (for DEFPASSWORD) |
| USER | string | Username (for DEFPASSWORD) |
| LISTENER_PORT | int | TCP port for socket server. **Must be 8500-8600** |

> Note: Spelling is `AUTHENTIFICATION` (not `AUTHENTICATION`) per DDF convention.

REST OAuth fields (runtime state, stored in GENERAL):

| Field | Type | Description |
|-------|------|-------------|
| CLIENT_ID | string | OAuth client ID |
| CLIENT_SECRET | string | OAuth client secret |
| REDIRECT_URI | string | OAuth redirect URI |
| ACCESS_TOKEN | string | Current access token |
| REFRESH_TOKEN | string | Refresh token |
| EXPIRES_IN | int | Token TTL in seconds |
| EXPIRES_DATE | int | Token expiry as epoch |
| TOKEN_TYPE | string | Token type (e.g., `Bearer`) |
| SCOPE | string | OAuth scope |

MODBUS-specific:

| Field | Type | Description |
|-------|------|-------------|
| PORT | int | Modbus port (default 502) |
| BAUDRATE | int | Baud rate |
| DATABITS | int | Data bits |
| STOPBITS | int | Stop bits |
| PARITY | enum | `NONE`, `ODD`, `EVEN` |
| FLOWCONTROL | enum | `NO`, `SW`, `HW` |
| TIMEOUT | int | Timeout in ms |
| WAIT_AFTER_CONNECT | int | Wait time in seconds |
| MIN_PAUSE | int | Minimum pause in ms |
| WORDORDER | enum | `BIG_ENDIAN`, `LITTLE_ENDIAN` |
| BYTEORDER | enum | `BIG_ENDIAN`, `LITTLE_ENDIAN` |
| ERROR_HANDLING | enum | `FLUSH_OR_RECONNECT_ON_ERROR`, `NOP_ON_ERROR`, `FLUSH_OR_CLOSE_ON_ERROR` |

SERIAL-specific:

| Field | Type | Description |
|-------|------|-------------|
| BAUDRATE | int | Baud rate |
| DATABITS | int | Data bits |
| STOPBITS | int | Stop bits |
| PARITY | enum | `NONE`, `ODD`, `EVEN` |
| FLOWCONTROL | enum | `NO`, `SW`, `HW` |
| WAIT_AFTER_SEND | int | Wait time in seconds after send |
| MIN_PAUSE | int | Minimum pause in ms |
| WORDORDER | enum | `BIG_ENDIAN`, `LITTLE_ENDIAN` (byte telegram only) |
| BYTEORDER | enum | `BIG_ENDIAN`, `LITTLE_ENDIAN` (byte telegram only) |

Other GENERAL fields:

| Field | Type | Description |
|-------|------|-------------|
| CONFIGFILE | `ENABLED` | Enable config file upload/download |

### `*COMMAND`

User-triggered commands for the DeviceStation UI.

| Column | Type | Description |
|--------|------|-------------|
| ID | int | Command index |
| ALIAS | string | Command name |
| FORMULA | string | Script to execute when command is triggered |

### `*READ` / `*PREPROCESS`

Polling-based communication methods. Executed at interval.

**REST protocol:**

| Column | Type | Description |
|--------|------|-------------|
| ALIAS | string | Unique alias for variable access |
| METHOD | string | `GET`, `POST`, `PATCH`, `PUT`, `DELETE`, `SSE` (Server-Sent Events), `LISTEN` |
| URL | string | URL path (combined with DOMAIN) |
| DATATYPE | string | `JSON`, `XML` |
| FORMULA | string | Code executed after method call |
| POLLING | int | Query interval in ms |

**MODBUS protocol:**

| Column | Type | Description |
|--------|------|-------------|
| ALIAS | string | Unique alias |
| FUNCTION | string | `FC1`..`FC16` (Modbus functions) |
| SLAVE | string | Slave address (number or variable, optional) |
| ADDRESS | int | Start address |
| COUNT | int | Number of registers |
| POLLING | int | Query interval in ms |

**Serial (SERCOM) protocol:**

| Column | Type | Description |
|--------|------|-------------|
| ALIAS | string | Unique alias |
| SEND | string | Send telegram |
| SEND_DELIMITER_START | string | Start marker (text or hex `/000203`) |
| SEND_DELIMITER_END | string | End marker |
| RECEIVE_LEN | int | Receive telegram length |
| DELIMITER_START | string | Receive start marker |
| DELIMITER_END | string | Receive end marker |
| POLLING | int | Query interval in ms |

> Note: `*READ` is empty in both REST pilot DDFs. All data fetching uses `*WRITE` with trigger flags.

### `*WRITE` / `*ONCHANGE`

Event-triggered communication. Executed on change/force. No polling interval.

Same column structure as `*READ` for each protocol, minus POLLING.

#### Inline `*ARGS` (sub-section of READ/WRITE/LISTENER)

Row format: first column empty, `*ARGS` or `ARGS` in second column.
Order of ARGS rows determines telegram structure.

**REST protocol ARGS:**

| Column | Type | Description |
|--------|------|-------------|
| METHOD | string | Always `ARGS` |
| ALIAS | string | Parent READ/WRITE alias (or `*ALL` for all methods) |
| TYPE | string | `url` (URL path segment), `arg` (query param), `data` (body param), `header` |
| NAME | string | Parameter name |
| VALUE | string | Static default value |
| ITEM | string | Dynamic value reference (e.g., `$.CONFIG.1`, `$SLAVE`, `X.10`) |
| FORMAT | string | Format string — see below |

**FORMAT values:**

| Format | Description |
|--------|-------------|
| `STRING` | String substitution (`%s` in NAME) |
| `DEZIMAL` | Integer substitution (`%lld` in NAME) |
| `DOUBLE` | Float substitution (`%d` in NAME) |
| `ISO8601` | Time as ISO 8601 string |
| `DAY` | Date as `YYYY-MM-DD` |
| `T_YYYYMMDDHHMM` | Time as YYYYMMDDHHMMSS DINT |
| `ENUM:val1=CODE1,...` | Integer-to-text mapping |
| `Bearer %s` | Custom format string with substitution |

**Special ARGS for JSON/XML structure:**

`BLOCK_START` / `BLOCK_END` — wraps child params in a JSON object
`ARRAY_START` / `ARRAY_END` — wraps child params in a JSON array

### `*LISTENER`

Socket-based event reception. Same structure as READ/WRITE per protocol.
Uses LISTENER_PORT from GENERAL.

### `*ITEM`

Logical variable definitions. The core data model of a DDF.

| Column | Type | Description |
|--------|------|-------------|
| ALIAS | string | Identifier for Plug&Play device assignment |
| NAME | string | Display name in the control system |
| ID | int | Unique ID 0..254. Referenced as `X.{ID}` in formulas |
| VISIBILITY | enum | `HIDDEN` or `VISIBLE` (default) |
| UNIT | string | Unit of measurement |
| GROUP | string | Comma-separated group IDs (e.g., `0,1,2,3`) |
| TYPE | enum | Empty=standard, `ARRAY`=REST JSON array, `STRING`=REST string |
| DEFAULT | string | Default value |
| WFORMULA | string | Write formula — executed on value change from control system |
| RFORMULA | string | Read formula — executed cyclically per POLLING |
| POLLING | int | Calculation/query interval in ms |
| MASK | string | Filter mask for IO selection per slave (compared with `$.ITEMMASK`) |

> **TYPE special values (DeviceLib.pdf p.14):**
> - `ARRAY`: RFORMULA contains array variable name. Query returns value at given index.
> - `STRING`: RFORMULA contains variable name for direct string access.
> - Data types for Gateway function: `bool`, `byte`, `word`, `dword`, `lword`, `usint`, `uint`, `udint`, `ulint`, `sint`, `int`, `dint`, `lint`, `real`, `lreal`, `string`, `time`

### `*GROUP`

Sub-device grouping. A device can have multiple sub-devices (e.g., battery, grid, PV).

| Column | Type | Description |
|--------|------|-------------|
| ID | int | Group index |
| ALIAS | string | Group identifier (used for Plug&Play matching) |
| NAME | string | Display name |

> Note: Column order varies between DDFs (ID;ALIAS;NAME vs. ALIAS;ID;NAME).
> Items can belong to multiple groups. Same ALIAS can have different values per group.

### `*OBJECT`

Generic device UI element definitions. Links ITEMs to display widgets.

| Column | Type | Description |
|--------|------|-------------|
| GROUP | int | Parent group ID |
| ID | int | Object index within group (0..19) |
| ALIAS | string | Display name (multilingual: comma-separated) |
| TYPE | int | Widget type — see table below |
| ENUM | int | Number of enum values (0..n) |
| ENUMTEXT | string | Comma-separated enum labels |
| ENUMVAL | string | Comma-separated enum values (values != -3) |
| MIN | float | Min value (for trends and input) |
| MAX | float | Max value (for trends and input) |
| IOTYPE | int | Currently unused |
| DIGITS | int | Decimal places for value display |
| ITEMID | int | Referenced ITEM ID |
| UNIT | string | Display unit |
| ALARM | string | Alarm comparison: `>`, `>=`, `<`, `<=`, `!=`, `==` |
| ALARMVAL | string | Alarm threshold value |
| ALARMTIME | int | Alarm delay in seconds |
| OUTTYPE | int | Currently unused |
| CMDITEMID | int | Command item ID reference |
| COMMAND | int | Number of commands |
| COMMANDENUM | string | Command enum labels (comma-separated) |
| COMMANDVAL | string | Command enum values (comma-separated, values != -3) |
| VIEWTYPE | int | Display type — see table below |
| LOG | int | Logging enabled (0/1) |

**TYPE values:**

| Value | Meaning |
|-------|---------|
| 1 | DEZIMAL / ENUM (boolean-like) |
| 2 | FLOAT (numeric value) |
| 3 | TEXT (string display) |
| 4 | WERT (gauge/value display) |
| 5 | URL (link) |
| 101 | // AMBIGUOUS — undocumented |
| 103 | // AMBIGUOUS — undocumented |
| 104 | Slider (numeric input) |

**VIEWTYPE values:**

| Value | Meaning |
|-------|---------|
| 0 | MAIN — visible in actions, widgets, and command selection |
| 1 | STATUS — status display |
| 2 | PARAMETER — parameter view |
| 4 | Single value as status in device menu |

> **DDF Graphics (DeviceLib.pdf p.19-20):** OBJECT can have additional columns `SVG`, `SVGANIMATE`, `SVGEVENT` for custom UI rendering. Sprint 0 does not implement graphic parsing.

> **Min/Max and logging:** If both MIN and MAX are 0, range 0-100 is assumed. If no log exists, trend is hidden.

### `*IO`

I/O signal definitions for the Generic Device system. Max 10 IO signals per group.
Access in formulas: `IO.GRP.ID`

| Column | Type | Description |
|--------|------|-------------|
| GROUP | int | Group ID reference |
| ID | int | Sequential ID (0, 1, 2, ...) |
| ALIAS | string | Signal name |
| DIR | enum | `DI` (digital in), `DO` (digital out), `AI` (analog in), `AO` (analog out) |

### `*CONFIG`

User-configurable parameters. Max 3 values. Retained across restarts.
Access in formulas: `$.CONFIG.ID` (read-only in formulas — `$.CONFIG.0 := 1` has no effect).

| Column | Type | Description |
|--------|------|-------------|
| ID | int | Config index (0, 1, 2) |
| ALIAS | string | Config field name |
| TYPE | int | 0 = TEXT, 1 = PASSWORD |

### `*FUNCTION` (MODBUS only)

Function definitions for MODBUS protocol. Allows defining reusable data copy operations.

| Column | Type | Description |
|--------|------|-------------|
| ALIAS | string | Function name |
| FORMULA | string | Function body |

Called via: `FUNCTION(SLAVE, TARGET, GET/SET, SOURCE, ADDRESS, COUNT)`

## Formula Language Reference

Per DeviceLib.pdf p.21-28. DDF formulas are an imperative scripting language.

### Syntax Rules

- Formulas must be enclosed in `"` quotes (because they contain `;`)
- Last character inside quotes must be `;`
- Comments: `/* this is a comment */` (multi-line allowed)
- No spaces allowed inside function calls
- No arithmetic operators (`+`, `-`, `/`, `*`) inside function parameters

### Variable Access

| Pattern | Description |
|---------|-------------|
| `X.{ID}` | Item variable by numeric ID |
| `ITEM.ALIAS` | Item variable by alias |
| `ITEM.GRPID.ALIAS` | Item variable by group + alias |
| `$.SYS.TIME` | System time in seconds |
| `$.SYS.TIME_MS` | System time in milliseconds |
| `$.PARAM.xxx` | Sub-device parameters |
| `$.GPARAM.xxx` | Main device parameters (e.g., `$.GPARAM.MANUFACTURER`) |
| `$.CONFIG.ID` | Config field value (read-only) |
| `$.SLAVE.x.yyy` | Cross-slave variable access |
| `$.GPARAM.URL` | URL for app link display |
| `TEMPVAR` | Temporary variable (device-scoped, auto-created) |

### Communication Method Variables

| Pattern | Description |
|---------|-------------|
| `ALIAS.F` | Force/Changed trigger (GET/SET) |
| `ALIAS.L` | Data length |
| `ALIAS.Q` | Quality |
| `ALIAS.EC` | Error counter (GET/SET) |
| `ALIAS.EN` | Error number |
| `ALIAS.T` | Timestamp (GET/SET) |
| `ALIAS.P` | Pause for x seconds (GET/SET) |

### REST JSON Access

| Pattern | Description |
|---------|-------------|
| `ALIAS.VALUE.path.to.field` | JSON object value |
| `ALIAS.VALUE.path[0].field` | JSON array element |
| `ALIAS.ARRAY.MEDIA.path` | Array average (Mittelwert, NOT median) |
| `ALIAS.ARRAY.MAX.path` | Array maximum |
| `ALIAS.ARRAY.MIN.path` | Array minimum |
| `ALIAS.ARRAY.LEN.path` | Array length |
| `ALIAS.FIND.STRING.path` | String search in object value |
| `ALIAS.ASLIST.path[].field` | Array values as comma-separated list |
| `ALIAS.EXIST.path` | Returns 1 if JSON/XML item exists |
| `ALIAS.URL` | URL string of current call |
| `ALIAS.HTTP_CODE` | HTTP response code |
| `ALIAS.HTTP_DATA` | Raw HTTP response data |

### Operators

| Operator | Meaning |
|----------|---------|
| `*` | Multiply |
| `/` | Divide |
| `+` | Add |
| `-` | Subtract |
| `^` | Power |
| `>`, `>=`, `<`, `<=` | Comparison |
| `!=` | Not equal |
| `==` | Equal |
| `>>`, `<<` | Bit shift right/left |
| `&&` | Logical AND |
| `&!` | AND NOT |
| `\|\|` | Logical OR |
| `\|` | Bitwise OR |
| `&` | Bitwise AND |
| `&~` | Bitwise AND NOT |
| `:=` | Assignment |

### Control Flow

```
IF condition THEN
    statements;
ELSE IF condition THEN
    statements;
ELSE
    statements;
ENDIF;
```

```
SWITCH variable
CASE value1:
    statements;
CASE value2:
    statements;
DEFAULT:
    statements;
ENDSWITCH;
```

```
FOR x := 1 TO y BY 1 DO
    statements;
ENDFOR;
```

## Plug&Play Device Aliases

DeviceLib.pdf p.30-35 defines standard ALIAS names for automatic device-to-function binding.
The control system searches for these aliases in the device's ITEMs.

Key element types and their expected aliases:

- **HEMS_GRID**: POWER_IMPORT, POWER_EXPORT, ENERGY_IMPORT, ENERGY_EXPORT, QUALITY, ERROR, INFO
- **HEMS_BATTERY**: POWER_CHARGE, POWER_DISCHARGE, SOC, QUALITY, ERROR, INFO
- **HEMS_PV**: POWER, ENERGY, QUALITY, ERROR, INFO
- **ENERGYMETER**: POWER, ENERGY, VOLTAGE_L1/L2/L3, CURRENT_L1/L2/L3, FREQUENCY
- **WEATHER-STATION**: WIND, RAIN, TEMPERATURE, DAYLIGHT, SUN_EAST/WEST/SOUTH
- **EMOBIL**: CURRENT_SET, POWER, ENERGY, PHASES, CPS, VOLTAGE/CURRENT per phase
- **ACCESS_DOOR**: UID, OPEN, BLOCKED, ENABLED, PERSONID, AUTHORIZED, DENIED
- **VENTILATION/Clima**: TEMP_AIR_SUPPLY/EXHAUST/OUTSIDE, HUMIDITY, AIR_QUALITY, OPERATING_MODE

Full list: see DeviceLib.pdf p.31-35.
